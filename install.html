下载Go发布版本

下载Go 点击访问下载页

官方二进制发布版 可用在 FreeBSD (8-STABLE发布版本及以上), Linux, Mac OS X (10.7版本及以上), 和 Windows 操作系统 以及 x86-32 和 x86-64 处理器架构.

如果二进制发布版本不适用当前系统和架构上, 可试试 源码安装 或者 安装gccgo编译器来代替gc编译器.

系统要求

Go二进制发布版只支持这些系统和架构. 在安装之前, 请确认操作系统满足这些要求. 如果操作系统或架构不在下面的列表中, 可能需要试一下 源码安装 或者 使用gccgo代替.

操作系统 架构 注意
FreeBSD 8-STABLE版本及以上 amd64 不支持Debian GNU/kFreeBSD
带glibc的Linux 2.6.23版本及以上 amd64, 386, arm 不支持CentOS/RHEL 5.x
Mac OS X 10.7版本及以上 amd64 use the clang or gcc† that comes with Xcode‡ for cgo support
Windows XP及以上 amd64, 386 使用MinGW gcc†.不需要cygwin or msys.

†gcc是必须的倘若计划使用 cgo.
‡仅需要为 Xcode安装命令行工具.如果你已经安装了Xcode 4.3+, 可以通过下载参数面板的 组件标签安装它.

安装Go tools

如果从老版本升级的话, 要先 卸载当前版本.

Linux, Mac OS X, and FreeBSD 源码

下载安装包并解压到/usr/local目录下, 这将会创建一个go树目录结构在 /usr/local/go目录中. 例如: 

tar -C /usr/local -xzf go$VERSION.$OS-$ARCH.tar.gz

选择合适的安装包. 比如, 如果要在64位linux上面安装1.2.1版本的Go, 则需要下载 go1.2.1.linux-amd64.tar.gz.

通常, 这些命令必须通过root用户或使用sudo命令执行.

使用下面命令将/usr/local/go/bin加到PATH环境变量里. 或者将下面的命令添加到/etc/profile文件中（系统级的安装）或$HOME/.profile文件里. 

export PATH=$PATH:/usr/local/go/bin

安装到自定义位置

Go二进制发布版本默认安装在 /usr/local/go目录(Windows下安装在 c:\Go目录), 也可以将Go tools安装在其他目录. 这种情况下你必须设置GOROOT环境变量 来指定Go tools所安装的位置.

例如, 将Go安装在Home目录下则需要将下面命令添加到 $HOME/.profile文件中: 

export GOROOT=$HOME/go
export PATH=$PATH:$GOROOT/bin

注意: 安装在自定义位置时，GOROOT必须设置.

Mac OS X软件包安装

下载安装包文件, 打开它, 根据提示安装Go tools. 软件包将Go默认安装在/usr/local/go.

软件包会将/usr/local/go/bin目录添加到 PATH环境变量中. 需要重启终端让设置生效.

Windows

Go工程为Windows用户提供了两种安装方式 (除了从源码安装外): 一种通过压缩包并且设置环境变量的方式和一种通过MSI安装程序自动安装配置.

MSI安装程序

打开MSI文件 根据提示安装Go tools. 安装程序默认将Go安装在c:\Go目录下.

安装程序会将c:\Go\bin目录添加到 PATH环境变量中.需要重启命令窗口来使设置生效.

Zip archive

下载zip文件并将其解压到指定目录(推荐 c:\Go目录).

倘若选择了非c:\Go目录, 则必须将指定的目录添加到GOROOT环境变量中.

将Go根目录下的bin子目录(例如, c:\Go\bin)添加到 PATH环境变量中.

在Windows下设置环境变量

Windows下设置环境变量：系统属性->高级->环境变量.

测试安装是否正确

通过设置工作空间并构建一个小程序来检查Go是否被正确安装.过程如下

创建一个包含工作空间的目录, $HOME/work 例如%HOME%\work ，并设置GOPATH环境变量指向该位置.命令如下： 

$ export GOPATH=$HOME/work

C:\> set GOPATH=%HOME%\work

将上面的命令添加到启动脚本中 (例如$HOME/.profile). 在Windows中，通过说明设置 GOPATH环境变量.

然后在工作空间中创建 src/github.com/user/hello目录 (如果使用GitHub,用你的GitHub用户名替换user), 并在hello目录中创建一个以hello.go命名的文件， 并在其中添加如下代码: 

package main

import "fmt"

func main() {
    fmt.Printf("hello, world\n")
}

使用go tool编译它: 

$ go install github.com/user/hello

C:\> go install github.com/user/hello

上面的命令将会把可执行文件hello (or hello.exe)移到bin工作目录中. 执行命令将看到hello world的问候: 

$ $GOPATH/bin/hello
hello, world

C:\> %GOPATH%\bin\hello
hello, world

如果看到hello,world的信息，说明Go安装成功.

在着急写代码之前看看 How to Write Go Code文档, 这个文档描述了一些关于Go tools的一些深入的概念.

卸载Go

把 go目录删除将会卸载Go. 通常Go目录是/usr/local/go Linux, Mac OS X,和FreeBSD这些系统下或者Windows下的c:\Go目录.

也应该移除bin目录从环境变量 PATH中. 在Linux和FreeBSD下编辑 /etc/profile文件或 $HOME/.profile文件将与Go相关的export命令删除. 如果使用 Mac OS X软件包安装Go则需要从 /etc/paths.d/go文件中删除相关路径. Windows用户参考关于 环境变量的设置.
